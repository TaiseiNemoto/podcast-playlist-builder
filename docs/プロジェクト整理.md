## 1. やりたいこと
田中渓さんのラジオ番組 「Beyond K-point」 において、  
過去の全エピソードで紹介された全ての曲を収集し、  
自分が利用している Apple Music のプレイリストに登録したい。

最終的には完全自動も視野に入れるが、まずは 安全性を優先した半自動運用 で始めたい。

---
## 2. 前提として分かっていること
- 番組で紹介された曲は、各エピソードの Apple Podcasts または Spotify の概要欄で確認できる
- 曲情報は毎回紹介されている想定だが、概要欄の書き方は完全統一とは限らない
- 最終的な登録先は Apple Music のプレイリスト
- 目的は、過去回を含む全放送回の紹介曲を網羅的に集めること
- Spotify Web API は Spotify Premium 前提で利用できなかったため、エピソード取得元は RSS を優先する

---
## 3. 実現方針
1. エピソード取得
	1. 番組 RSS から全エピソードのタイトル・概要欄・公開日を取得
2. 曲抽出
	1. 概要欄から曲名・アーティストを抽出
3. 一覧化＋確認＋登録
	1. CSVで一覧化 → Apple Music検索結果と突合
	2. 確認OKの曲のみAPIでプレイリストに追加

---
## 4. 技術方針
- 言語：Python
- データ取得：Podcast RSS（エピソード＋概要欄）
- 出力：CSV（確認用）
- Apple Music：
    - 検索：Catalog API
    - 登録：Library Playlist API
- 認証：
    - RSS取得フェーズでは不要
    - Apple Music検索では Developer Token を利用
    - プレイリスト作成・曲追加では Music User Token を利用
    - Music User Token の取得は MusicKit on the Web を使ったローカルWeb画面で行う

---
## 5. フェーズ1の開発方針
1. 番組 RSS URL を特定する
2. RSS から全エピソードの以下を取得する
	1. タイトル
	2. 公開日
	3. 概要欄
	4. エピソードURL
3. 取得結果を JSON で保存する
4. 後続の曲抽出で使いやすいデータ構造に整える

---
## 6. フェーズ3の基本方針
1. Python だけで完結させない
	1. Apple Music の検索候補生成は Python で行う
	2. ユーザー認証とプレイリスト操作はローカルWeb画面で行う
2. 自動登録ではなく半自動にする
	1. 検索候補を CSV で一覧化する
	2. 人が確認して採用可否を決める
	3. 確認 OK の曲だけを Apple Music に登録する
3. プレイリスト作成は今回 1 回のみとする
	1. 最初に 1 つのプレイリストを作成する
	2. その後はそのプレイリストへ承認済み曲を追加する

---
## 7. フェーズ3の実装イメージ
1. Apple Music 候補検索
	1. 入力：`data/tracks.csv`
	2. 処理：Apple Music Catalog API で曲候補を検索
	3. 出力：`data/apple_music_candidates.csv`
2. 候補確認
	1. 候補 CSV を目視確認する
	2. 採用する候補に `approved` と `selected_apple_music_id` を記入する
	3. 確認結果ファイルを `data/apple_music_matches.csv` として保存する
3. プレイリスト作成＋曲追加
	1. ローカルWeb画面で Apple Music 認証を行う
	2. 初回のみプレイリストを 1 つ作成する
	3. `approved=1` の曲だけをプレイリストに追加する
	4. 追加結果をログまたは CSV で保存する

---
## 8. Apple Music 連携の仕様前提
1. Catalog API の検索は Developer Token で実行する
2. ライブラリ操作は Music User Token が必要
3. プレイリスト作成・曲追加は `me/library` 系 API を使う
4. Developer Token は Apple Developer 側で発行する JWT を使う
5. Music User Token は MusicKit on the Web でユーザー認証して取得する

---
## 9. フェーズ3-1の実装計画
1. `scripts/search_apple_music.py` を作成する
2. `data/tracks.csv` を読み込む
3. 各曲について Apple Music Catalog API を検索する
	1. まず `artist + title` で検索
	2. 必要に応じて `title + artist` や簡易正規化版でも検索
4. 上位候補を 3 件から 5 件出力する
5. 候補ごとに以下を CSV へ保存する
	1. 元の `artist`
	2. 元の `title`
	3. 候補順位
	4. Apple Music の song id
	5. 候補の曲名
	6. 候補のアーティスト名
	7. アルバム名
	8. 簡易スコア
	9. 採用確認用の列

---
## 10. フェーズ3-2の実装計画
1. `apple_music_candidates.csv` を人が確認する
2. 採用候補だけを `approved=1` にする
3. 最終採用する `selected_apple_music_id` を 1 件だけ残す
4. その結果を登録用入力として使う

---
## 11. フェーズ3-3の実装計画
1. ローカルWeb画面を用意する
2. MusicKit on the Web で Apple Music ログインを行う
3. Music User Token を取得する
4. 初回のみ新規プレイリストを 1 つ作成する
5. `apple_music_matches.csv` の承認済み song id を順次追加する
6. 成功・失敗・重複スキップを結果として保存する

---
## 12. 注意点
1. 検索結果の自動採用は行わない
2. 表記揺れや Remix 表記があるため、人確認を必須とする
3. Apple Music 未配信の曲がある可能性を前提にする
4. 1 回きりのプレイリスト作成であるため、まずは「新規作成専用」で実装する
