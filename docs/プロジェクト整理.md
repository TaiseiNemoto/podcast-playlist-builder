## 1. やりたいこと
田中渓さんのラジオ番組 「Beyond K-point」 において、  
過去の全エピソードで紹介された全ての曲を収集し、  
自分が利用している Apple Music のプレイリストに登録したい。

最終的には完全自動も視野に入れるが、まずは 安全性を優先した半自動運用 で始めたい。

---
## 2. 前提として分かっていること
- 番組で紹介された曲は、各エピソードの Apple Podcasts または Spotify の概要欄で確認できる
- 曲情報は毎回紹介されている想定だが、概要欄の書き方は完全統一とは限らない
- 最終的な登録先は Apple Music のプレイリスト
- 目的は、過去回を含む全放送回の紹介曲を網羅的に集めること
- Spotify Web API は Spotify Premium 前提で利用できなかったため、エピソード取得元は RSS を優先する

---
## 3. 実現方針
1. エピソード取得
	1. 番組 RSS から全エピソードのタイトル・概要欄・公開日を取得
2. 曲抽出
	1. 概要欄から曲名・アーティストを抽出
3. 一覧化＋確認＋登録
	1. CSVで一覧化 → Apple Music検索結果と突合
	2. 確認OKの曲のみAPIでプレイリストに追加

---
## 4. 技術方針
- 言語：Python
- データ取得：Podcast RSS（エピソード＋概要欄）
- 出力：CSV（確認用）
- Apple Music：
    - 検索：カタログAPI
    - 登録：プレイリスト追加API
- 認証：
    - RSS取得フェーズでは不要
    - Apple Music連携フェーズで Developer Token / Music User Token を利用

---
## 5. フェーズ1の開発方針
1. 番組 RSS URL を特定する
2. RSS から全エピソードの以下を取得する
	1. タイトル
	2. 公開日
	3. 概要欄
	4. エピソードURL
3. 取得結果を JSON で保存する
4. 後続の曲抽出で使いやすいデータ構造に整える
